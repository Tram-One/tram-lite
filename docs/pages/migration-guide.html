Migration Guide

This page goes over all the major breaking changes between different versions of Tram-Lite. While upgrading is never required, if you want to make use of newer features in your old projects, this should help! If you need any help following this guide, or if there is a case that isn't covered, feel free to join the discord and reach out!

Breaking changes in Version 5

If you are upgrading from version 4, here are the key breaking changes you'll need to fix to go to version 5

Removed Node Support. If you have a compelling use case here, please reach out!

Removed shadowRootProcessors on base HTMLElement. If you were using a shadowRootProcessor on a non-Tram-Lite component (like tl-effect or tl-controlled) these will no longer work by default. In order to keep this behavior, you can run TramLite.appendShadowRootProcessor('[tl-effect]', ComponentEffect, HTMLElement);

Changed tl-hostattr and tl-targetattr to tl-attrmap. This should allow users to map multiple attributes in a single input. You will not be able to map attribute with a : in their name, if that was something you were doing before.

Breaking changes in Version 4

If you are upgrading from version 3, here are the key breaking changes you'll need to fix to go to version 4

Places where you had should now be

Script tags now need the attribute tl-effect. Script tags that referenced external sources will no longer work. See this github issue.

Removed html and svg functions. There are no substitutes for this API in v4. If you still need this functionality, you can recreate it by creating an element and setting the innerHTML.

Top-level functions like define, updateRootAttr, and addAttributeListener were moved to static methods in the TramLite class. You should be able to call the new methods off the TramLite class, e.g. TramLite.define.
